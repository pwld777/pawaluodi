# 《花儿与少年》音乐课堂游戏核心玩法升级改进计划

> 目标不是推倒重做，而是在保留“三个游戏核心玩法”的基础上，把它升级成更像 4399 小游戏 / 儿童网页小游戏的体验：更有任务感、更强反馈、更专业的音乐知识、更清楚的排版和视觉层级。

## 1. 当前问题诊断

### 1.1 最大问题不是“素材不够”，而是“游戏包装层缺失”

当前版本已经能玩，但体验更像一个课堂交互原型：

- 首页像说明页，不像关卡大厅。
- 三个游戏只是并列模块，缺少连续闯关、奖励、进度、过场和完成感。
- 图像、按钮、谱例、说明文字放在同一层级，视觉像“堆组件”。
- 操作反馈偏弱，学生点对/点错后缺少小游戏常见的“爽感”。
- 音乐知识被做成了简单规则，但没有足够明确地服务《花儿与少年》的段落、节拍、节奏和风格对比。

### 1.2 保留的核心玩法

这三个核心玩法应保留，不重做：

1. **花鼓节拍挑战**：学生通过鼓心/鼓边体验强拍、弱拍，练习 2/4 与 3/4。
2. **节奏拖拽选择**：学生根据听辨/提示，把正确节奏卡拖入答案区。
3. **拼图创编工坊**：学生选择节拍、节奏型、乐器音色，完成小节创编。

升级方向是：把这三个玩法包装成“关卡制小游戏”，而不是换一套完全不同机制。

## 2. 外部参考结论

### 2.1 4399 / 儿童网页小游戏可借鉴点

4399 的页面和小游戏目录本身有很强的“即点即玩、分类清楚、儿童/休闲/过关入口明确”的特征。4399 首页将游戏按动作、益智、休闲、儿童、过关等分类展示，学生进入前就能感到这是一个“游戏入口”而不是课件列表。[4399 首页](https://www.4399.com/)也明确呈现“儿童小游戏”“过关”等分类。

从 4399 儿童音乐小游戏和音乐手游介绍可提炼出几个适合本项目的原则：

- **进入即任务**：例如 4399 儿童音乐小游戏会用简单说明告诉孩子“听音乐感受情绪”，目标非常短。[小老虎感受音乐](https://www.4399.com/flash/97629.htm)
- **操作低门槛**：音乐节奏游戏常用点击、长按、滑动、跟随节奏等简单动作，重点考反应与节拍。[指尖钢琴 4399 介绍](https://a.4399.cn/game-id-228122.html)
- **新手引导明确**：音乐舞蹈类游戏会先用 NPC 或教程带玩家学习基本按键。[4399Touch 新闻](https://news.4399.com/touch/xinwen/201307-04-277873.html)
- **打击感要强**：音乐游戏强调“随着音乐节拍跳动”“节奏感和打击感”。[Touch 触动](https://my.4399.com/yxtouch/)

本项目不应该照搬 4399 的商业页游风，而应借鉴它的“入口明确、按钮大、任务短、反馈强、闯关感强”。

### 2.2 Game studio / 休闲游戏设计可借鉴点

专业游戏设计里，最重要的是核心循环。Roblox Creator Hub 将核心循环拆成玩家反复进行的行动，并强调需要 progression engine 推动玩家继续体验，否则会重复、无聊、浅。[Roblox Core Loops](https://create.roblox.com/docs/production/game-design/core-loops)

GameAnalytics 对 core loop 的建议也强调：玩法循环、进度系统、反馈与整体统一性要一起设计，反馈不是最后贴上去的装饰。[GameAnalytics Core Loop](https://www.gameanalytics.com/blog/how-to-perfect-your-games-core-loop)

GDevelop 对休闲游戏循环的解释可转化为本项目规则：

```text
看到任务 -> 做一个简单动作 -> 获得即时反馈 -> 得到奖励/解锁 -> 进入下一关
```

也就是每一关都要有一分钟内能反复感受到的循环。[GDevelop Casual Game Loops](https://gdevelop.io/ar-sa/blog/casual-game-loops)

Game feel / juice 资料强调，游戏的“好玩感”来自操作后的多层反馈：动画、音效、粒子、按钮压感、命中反馈、奖励翻牌等。它不是为了炫，而是让玩家知道“我刚才做对了什么”。[GameJuice MDA](https://gamejuice.co.uk/articles/mda-framework-rules-to-feelings)、[Game Feel Survey](https://arxiv.org/abs/2011.09201)

教育游戏方面，游戏化学习强调清晰目标、即时反馈、逐步挑战和反思环节。Quest to Learn 的游戏化学习原则中也包含“反馈即时且持续”。[Quest to Learn](https://en.wikipedia.org/wiki/Quest_to_Learn)

## 3. 音乐专业内容校准

### 3.1 需要先确认的音乐事实

当前游戏把《花儿与少年》简化成“第一段 2/4，第二段 3/4，变拍挑战”。这个方向大体可用，但必须更严谨：

- 教案资料显示，A 段旋律欢快活泼，二四拍，以八分音符和十六分音符交织而成；B 段与 A 段形成对比，旋律优美抒情，三四拍；第三段再现第一段旋律。[21 世纪教育网教案摘要](https://zy.21cnjy.com/12103981)
- 关于“花儿”音乐，资料指出其节奏具有多样性，常见八分音符、十六分音符、前八后十六等形态，不能只用“强弱拍”概括。[西北民歌“花儿”的艺术特点](https://www.boailunwen.com/fwsx/ys/2675.html)
- “花儿”本身与青海、西北民歌传统相关，情境不应只做成普通课堂或随便的高原背景，而应围绕“花儿会 / 青海民歌 / 小乐队表演 / 民族歌舞场景”。[中国非遗数字博物馆：青海花儿](https://www.ihchina.cn/news_1_details/11113.html)

### 3.2 音乐知识修正原则

1. **不再把游戏一写成“严格跟原曲判分”**  
   第一版网页不播放主音乐，也没有和教师音频同步，所以“变拍挑战”只能做练习模式或教师口令模式，不能假装能精准判定原曲变拍。

2. **A/B/A 结构要进入游戏叙事**  
   关卡地图应体现：A 段欢快花鼓队 -> B 段舒展高原旋律 -> A 段再现小舞台合奏。

3. **节奏题要从音乐特征出发**  
   节奏拖拽不能只问“跳动/舒展”，应围绕：
   - 2/4 强弱循环
   - 3/4 强弱弱循环
   - 八分音符与十六分音符形成的欢快感
   - 舒展段更长音值、更连贯的旋律感

4. **创编要服务风格对比**  
   创编工坊不是随便拼音符，而是要让学生拼出“欢快段”和“悠扬段”的差异。

## 4. 总体升级方向

### 4.1 不重做的边界

保留：

- 单页应用结构。
- 首页、花鼓、节奏、创编、展示五个视图。
- 底部导航。
- `localStorage` 状态。
- Web Audio 播放引擎，但音源要从合成波形升级为真实采样。
- 三个游戏的核心数据结构。

升级：

- 首页改成“关卡大厅”。
- 每个游戏改成一个独立小游戏界面，而不是普通教学卡片。
- 强化任务、倒计时、连击、星级、过关动画。
- 统一美术方向、素材风格和排版网格。
- 校准音乐知识和题目文本。

### 4.2 新核心循环

全局循环：

```text
进入花儿会大厅 -> 选择关卡 -> 完成小游戏挑战 -> 得星星/徽章 -> 点亮下一关 -> 展示小乐队作品
```

每关循环：

```text
看任务卡 -> 进行 3 到 6 次短操作 -> 即时反馈 -> 连击/星星 -> 小结一句音乐知识 -> 下一关
```

这能把“课堂任务”变成“小游戏关卡”。

## 5. 首页升级：从说明页到关卡大厅

### 5.1 当前问题

首页现在标题、说明、按钮、情景图、三张关卡卡片都在一个页面里，但没有明显主次。学生看不到“我要先玩哪个、玩完会怎样”。

### 5.2 升级方案

首页改为“花儿会闯关大厅”：

```text
顶部 HUD：花儿会闯关 / 星星数 / 今日任务

中心主视觉：青海花儿会小舞台
左侧：教师口令提示 / 玩法提示
右侧：学生小花鼓队角色或奖章墙

底部：三张大关卡牌
1 花鼓节拍台
2 节奏采花路
3 小乐队创编台
```

### 5.3 关卡牌信息

每张关卡牌必须有：

- 关卡编号。
- 主图标或角色。
- 一句话任务。
- 可得星星。
- 当前状态：未开始 / 进行中 / 已通关。
- 大按钮：`开始挑战` / `继续挑战` / `再玩一次`。

### 5.4 首页视觉标准

- 不再用“正文说明”解释所有玩法。
- 一屏内只保留 1 个主行动按钮。
- 三张关卡牌使用统一尺寸和对齐，不随内容长度乱跳。
- 背景用“青海花儿会舞台”而不是普通风景图直接铺满。
- 情景图要有 UI 留白，不能让标题压在人物脸上。

## 6. 游戏一升级：花鼓节拍挑战

### 6.1 保留玩法与音乐边界

第一关的音乐逻辑必须收窄并做准：它不是“随便敲鼓”，也不是复杂节奏创编，而是专门训练学生识别节拍强弱关系。

必须保留并强化：

- 鼓心 = 强拍。
- 鼓边 = 弱拍。
- 2/4 = 强 弱。
- 3/4 = 强 弱 弱。

第一关不要混入第三关的创编逻辑，也不要让学生自由拼节奏。第一关的核心评价只有一个：学生是否能在稳定拍点中识别并敲出 **强弱** 或 **强弱弱**。

### 6.2 当前问题

- 大鼓已经变真实，但玩法界面还像练习器。
- 点击对错反馈不够像音乐节奏游戏。
- 右侧提示和左侧步骤比较像课件说明。
- “变拍挑战”容易误导为可以精准同步教师主音乐。

### 6.3 玩法升级：花鼓节奏台

把游戏一包装成“太鼓式节奏台”，但保留鼓心/鼓边点击机制。

界面结构：

```text
左上：关卡名 花鼓节奏台
右上：分数 / 连击 / 星级

中间：真实大花鼓
鼓面上方：拍点轨道
轨道中圆点向判定圈移动

左下：当前识别模式
认识强弱 / 2拍子：强弱 / 3拍子：强弱弱 / A-B-A 对比练习

右下：音乐知识小卡
“2/4 像踏步：强 弱”
“3/4 像摇摆：强 弱 弱”
```

### 6.4 交互升级

1. **认识强弱**
   - 屏幕出现两步引导：点鼓心、点鼓边。
   - 点对后出现“强拍命中”“弱拍命中”徽章。
   - 点错不扣分，只给视觉提示。

2. **强弱识别：2/4**
   - 拍点轨道显示两个点循环：金色强拍、青色弱拍。
   - 学生必须按 `鼓心 鼓边` 循环点击。
   - 判定重点是顺序是否为 `强 弱`，时间窗口可以宽容。
   - 连续 4 次正确获得一颗星。

3. **强弱弱识别：3/4**
   - 轨道显示三个点循环：金色、青色、青色。
   - 学生必须按 `鼓心 鼓边 鼓边` 循环点击。
   - 鼓边弱拍反馈更轻，速度略慢，表现舒展感。
   - 判定重点是能否把第二、第三拍都识别为弱拍，不能把第三拍误判成强拍。

4. **A-B-A 对比练习**
   - 不再写“跟教师音乐精准变拍”。
   - 改为练习关：系统倒计时后先出现 2/4 的 `强弱`，再出现 3/4 的 `强弱弱`，最后回到 2/4。
   - 学生任务是看/听到模式变化后切换正确敲法。
   - 课堂正式使用时只显示“听到老师播放进入 B 段时，换成强弱弱”，判定宽容。

### 6.5 第一关判定标准

第一关必须有清楚的状态机，不要只靠文案：

| 模式 | 正确序列 | 错误类型 | 反馈 |
| --- | --- | --- | --- |
| 认识强弱 | 先鼓心，再鼓边 | 点反区域 | “强拍在鼓心 / 弱拍在鼓边” |
| 2/4 | 强 弱 | 把弱拍点成鼓心 | “这是弱拍，轻敲鼓边” |
| 3/4 | 强 弱 弱 | 第 3 拍点成鼓心 | “三拍子后两拍都是弱拍” |
| A-B-A | 强弱 -> 强弱弱 -> 强弱 | 切换慢或切错 | “B 段要换成强弱弱” |

技术实现建议：

- `beat-patterns.js` 中显式保存 `pattern: ["strong", "weak"]` 和 `["strong", "weak", "weak"]`。
- 判定函数不要只返回 `wrong-zone`，要返回 `wrong-third-beat`、`wrong-meter-pattern` 等更具体原因。
- UI 上强拍点和弱拍点颜色固定，避免学生把颜色和位置混淆。

### 6.6 反馈升级

必须加入：

- 鼓心正确：金色波纹、鼓面轻微下压、短促鼓声。
- 鼓边正确：青绿色外圈波纹、轻鼓边音。
- 错误：鼓面不红屏，使用“再听一听”橙色气泡。
- 连击：`Combo 2/3/4` 小数字飞起。
- 过关：星章翻面，花瓣从鼓边散出。

### 6.7 音乐知识升级

游戏一结束小结：

```text
A 段欢快活泼：2/4，强弱清楚，适合花鼓踏步。
B 段优美抒情：3/4，强弱弱有摇摆感。
再现 A 段：欢快节奏回来，小花鼓队重新热闹起来。
```

## 7. 游戏二升级：节奏拖拽选择

### 7.1 保留玩法

保留“拖拽正确节奏卡到答案框”。

### 7.2 当前问题

- 题目像练习题，不像小游戏。
- 谱例卡没有“收集/选择/命中”的手感。
- 音乐知识不够具体。
- 拖拽区域和卡片视觉层级不够强。

### 7.3 玩法升级：节奏采花路

把节奏题包装成“沿着高原花路采正确节奏花”。

核心不变：学生仍然拖正确卡片到答案区。

界面结构：

```text
左侧：题目情境卡
“A 段小花鼓队跑起来，哪张节奏最跳动？”

中间：花路答案槽
正确卡拖入后，花路点亮一段

右侧/底部：3 到 4 张节奏卡
每张是谱例卡 + 节奏读法 + 风格标签
```

### 7.4 题型升级

至少做 6 题，分为 3 类：

1. **节拍识别**
   - 2/4：强 弱
   - 3/4：强 弱 弱

2. **节奏疏密**
   - 八分音符 / 十六分音符组合表现欢快跳动。
   - 较长音值表现舒展悠扬。

3. **段落风格匹配**
   - A 段：欢快、活泼、跳动。
   - B 段：优美、舒展、抒情。
   - A 再现：欢快回归。

### 7.5 卡片设计

每张节奏卡包含：

- 谱例图形。
- 读法：`ta ta-ti` 或中文节奏口令。
- 拍数标签。
- 风格标签：跳动 / 舒展 / 稳定 / 密集。

注意：谱例不能随便画，需要和 `notation-cards.js` 的时值一致。

### 7.6 反馈升级

- 拖拽时卡片抬起，投影变大。
- 答案槽吸附，边框发光。
- 正确：花路开花一格，星星飞到 HUD。
- 错误：卡片弹回，提示“这张太舒展/太密集，再听 A 段的跳动感”。
- 完成：生成“节奏采花徽章”。

## 8. 游戏三升级：拼图创编工坊

### 8.1 保留玩法

保留：

- 选择节拍。
- 选择欢快段/悠扬段。
- 选择乐器。
- 把**标准节奏拼图块**放入小节。
- 满足小节拍数才算完成。
- 播放学生实际编创出的节奏。

明确不保留：

- 不把五线谱当主要创编界面。小学课堂里的这个游戏要做“节奏拼图”，不是让学生读五线谱。
- 不用装饰性五线谱假装专业。节奏块必须按照标准节奏时值组织。
- 不播放固定预设音乐冒充学生作品。播放结果必须来自学生拖进去的拼图顺序。

### 8.2 当前问题

- 更像表单和编辑器，不像游戏。
- 乐器虽然换成图片，但按钮排版还没有“乐器台”感觉。
- 创编反馈弱，学生不知道自己创作出了什么风格。
- 现有 2 小节太短，创编不像一个完整小乐句。
- 小节区域不该像五线谱编辑器，应更像节奏拼图槽。
- 当前播放逻辑应升级为“按每个节奏块内部节奏发声”，而不是每个块只响一下。
- 现在使用 Web Audio 合成音色，不符合“真实音色”要求。

### 8.3 玩法升级：小乐队创编台

界面包装成“小花鼓队排练室 / 创编台”。

结构：

```text
左侧：任务订单
“帮小花鼓队编 4 小节欢快段”
“要求：2/4，每小节装满 2 拍”

中间：节奏拼图板
4 个小节槽，每个小节有明确拍格
播放时光标逐音符移动

右侧：乐器台
真实乐器图片：花鼓、木鱼、铃鼓、沙锤

底部：节奏积木抽屉
```

### 8.4 标准节奏拼图规则

第三关的节奏块必须按标准节奏时值来写。建议不用五线谱作为主视觉，而用“大节奏拼图块 + 节奏符号 + 拍数 + 口令”的形式。

基础节奏块：

| 节奏块 | 拍数 | 2/4 可用 | 3/4 可用 | 发声规则 |
| --- | ---: | --- | --- | --- |
| 四分音符 `ta` | 1 拍 | 是 | 是 | 小节当前拍点响 1 下 |
| 两个八分音符 `ta-ti` | 1 拍 | 是 | 是 | 半拍响 2 下 |
| 四个十六分音符 `ta-ka-di-mi` | 1 拍 | 是 | 是 | 四分之一拍响 4 下 |
| 二分音符 `ta-a` | 2 拍 | 是 | 是 | 第 1 拍响，延长到 2 拍 |
| 附点二分音符 `ta-a-a` | 3 拍 | 否 | 是 | 第 1 拍响，延长到 3 拍 |
| 四分休止 `休` | 1 拍 | 是 | 是 | 占 1 拍但不发声 |

可扩展节奏块：

- 前八后十六：`ta-ti-ti`，1 拍，内部偏移 `[0, 0.5, 0.75]`。
- 前十六后二八：`ta-ka-ti`，1 拍，内部偏移 `[0, 0.25, 0.5]`。
- 两个四分音符：2 拍，内部偏移 `[0, 1]`。
- 三个四分音符：3 拍，只在 3/4 中作为完整小节块使用。

数据结构建议：

```js
{
  id: "four-sixteenths",
  label: "四个十六分音符",
  beats: 1,
  syllables: "ta-ka-di-mi",
  hits: [0, 0.25, 0.5, 0.75],
  sustainBeats: 0.15,
  moods: ["欢快", "密集"]
}
```

这里 `hits` 是相对于节奏块开始位置的拍内偏移。播放时必须逐个 offset 调度真实采样。

### 8.5 小节数量与任务升级

不要只给一个自由工坊。改成三种任务：

1. **欢快段任务**
   - 节拍：2/4。
   - 小节数：至少 4 小节，进阶可 8 小节。
   - 推荐节奏：八分、十六分、短音型。
   - 推荐乐器：花鼓、铃鼓、沙锤。

2. **悠扬段任务**
   - 节拍：3/4。
   - 小节数：至少 4 小节，进阶可 8 小节。
   - 推荐节奏：长音型、较少切分。
   - 推荐乐器：木鱼轻敲、花鼓弱音或柔和沙锤。

3. **自由展示任务**
   - 学生可以选择任意节拍和乐器。
   - 系统不强行判错，只给风格评价。

小节槽显示规则：

- 2/4 每小节显示 2 个大拍格。
- 3/4 每小节显示 3 个大拍格。
- 八分和十六分节奏块放入后，在一个大拍格内部显示拆分，不新增小节容量。
- 拼图块必须“刚好卡满”小节，不允许溢出。
- 休止符也占拍数，播放时不发声。

### 8.6 音乐评价规则

创编完成后给三项评价，不只给“装满”：

- **拍数正确**：每小节刚好装满。
- **风格匹配**：欢快段是否有短音/密集节奏，悠扬段是否更舒展。
- **音色合适**：乐器是否与段落气质一致。

显示方式：

```text
节拍稳定 ★
风格清楚 ★
音色搭配 ★
```

### 8.7 播放升级：必须播放学生编创的节奏

当前播放只高亮小节。升级后必须按学生拼图数据生成真实节奏播放：

- 播放光标从左到右移动。
- 当前节奏积木逐个点亮。
- 节奏块内部的八分、十六分要按真实拍内位置逐下发声。
- 休止符不发声，但光标仍通过。
- 当前乐器图片轻轻震动。
- 每个音符触发小粒子。
- 播放结束出现“小乐队排练完成”。

技术实现建议：

```text
小节开始时间 = barIndex * beatsPerBar * beatMs
节奏块开始时间 = 小节开始时间 + blockStartBeat * beatMs
每个 hit 发声时间 = 节奏块开始时间 + hitOffset * beatMs
```

如果学生拼的是：

```text
2/4 第 1 小节：两个八分音符 + 四分音符
```

播放应为：

```text
第 0 拍响，第 0.5 拍响，第 1 拍响
```

而不是“这个小节响两下”或“每个卡片只响一下”。

第三关音色必须从合成音色升级为真实采样音色。
